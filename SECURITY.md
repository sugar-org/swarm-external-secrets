# Security Policy

## Supported Versions

Security fixes are applied to the `main` branch. Use the latest commit from `main` for the most up-to-date security patches.

## Reporting a Vulnerability

Please do **not** open public issues for sensitive vulnerabilities.

Report vulnerabilities by opening a private security advisory:

1. Go to the repository's **Security** tab.
2. Click **Report a vulnerability**.
3. Include:
   - impact and affected component(s)
   - reproduction steps or proof of concept
   - suggested remediation if available

If private advisory reporting is unavailable, contact maintainers through the project communication channels and request a private response path.

## Response Expectations

- Initial acknowledgment: within 5 business days.
- Triage and severity assessment: as soon as enough information is provided.
- Fix timeline: prioritized by severity and exploitation risk.

We follow responsible disclosure and coordinate release notes once a fix is available.
